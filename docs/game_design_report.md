## 1. 游戏概述

### 1.1 核心定位

OpenFoot Manager 是一款足球经理模拟游戏，玩家扮演足球俱乐部主教练，负责球队管理的方方面面：战术部署、球员转会、青训培养、财务运营，最终带领球队走向巅峰。

### 1.2 核心玩法循环

```
选择球队 → 制定战术 → 推进日程 → 比赛模拟 → 结果反馈 → 调整策略 → 循环
```

### 1.3 主要特性

| 特性 | 描述 |
|------|------|
| 多联赛支持 | 英超、西甲、德甲、意甲、法甲五大联赛 |
| 欧冠赛事 | 完整的欧洲冠军联赛模拟 |
| 世界杯 | 国际赛事体系 |
| 逐日推进 | 每天逐步推进，非回合跳跃 |
| 实时财务 | 完整的俱乐部经济模拟 |
| 球员成长 | 基于训练和比赛的球员发展 |
| AI管理 | 智能AI管理对手球队 |

### 1.4 赛季时间线

| 时间节点 | 事件 |
|----------|------|
| 8月10日 | 赛季开始 |
| 6月-8月 | 夏季转会窗口 |
| 1月 | 冬季转会窗口 |
| 每周一 | 工资支付、周报生成 |
| 每月1日 | 赞助/转播/商品/维护费结算、董事会评估 |

## 2. 技术架构

### 2.1 架构模式

采用 **Entity-System** 架构：

- **实体(Entity)**: 球员、球队、教练组等数据对象
- **系统(System)**: 无状态的逻辑处理模块
- **状态中心(GameState)**: 全局唯一的数据容器
- **事件总线(EventBus)**: 系统间解耦通信

```
┌─────────────────────────────────────────────────┐
│                   main.lua (入口)                 │
├─────────────────────────────────────────────────┤
│  Router (路由)  │  EventBus (事件)  │  UI层      │
├─────────────────────────────────────────────────┤
│              core/game_state.lua (状态中心)       │
├─────────────────────────────────────────────────┤
│  turn_processor │ match_engine │ finance │ ...  │
│              systems/ (系统层)                    │
├─────────────────────────────────────────────────┤
│  player.lua │ team.lua │ staff.lua │ ...        │
│              domain/ (领域模型)                   │
└─────────────────────────────────────────────────┘
```

```lua
-- 发布事件

-- 订阅事件
EventBus.subscribe("match_completed", function(data)
    MoraleManager.onMatchResult(data)
end)
```

```lua
-- 导航到画面
Router.navigate("squad", { teamId = myTeamId })

-- 带参数替换当前画面
Router.replaceWith("player_detail", { playerId = pid })
```

## 3. 核心游戏循环

### 3.1 日程推进机制

游戏采用**逐日推进**模式，每次玩家点击"下一天"按钮触发 `advanceDay()`:

```
advanceDay()
  ├── 检查是否为比赛日
  │   ├── 是 → processMatchDay() (模拟所有当日赛事)
  │   └── 否 → processNonMatchDay() (日常处理)
  ├── 检查是否为周一 → processWeekly()
  └── 检查是否为月初 → processMonthly()
```

### 3.2 每日处理 (Non-Match Day)

| 处理项 | 说明 |
|--------|------|
| 训练 | 执行当日训练计划，球员属性成长 |
| 伤病恢复 | 受伤球员恢复进度推进 |
| 体能恢复 | 非训练日体能自然恢复 |
| 合同检查 | 检查即将到期的合同 |
| 转会处理 | 处理进行中的转会报价 |
| 球探任务 | 球探执行发掘任务 |
| 随机事件 | 5%概率触发转会传闻等事件 |
| 青训 | 青年队日常发展 |

### 3.3 每周处理 (周一)

| 处理项 | 说明 |
|--------|------|
| 工资支付 | 支付全队周薪 |
| 伤病更新 | 可能产生新的训练伤 |
| 士气更新 | 基于多因素调整全队士气 |
| 声望更新 | 基于近期表现调整俱乐部声望 |
| AI管理 | AI球队执行转会/战术决策 |
| 新闻生成 | 生成周报类新闻 |

### 3.4 每月处理 (1号)

| 处理项 | 说明 |
|--------|------|
| 赞助商收入 | 结算月度赞助费 |
| 转播收入 | 结算月度转播分成 |
| 商品收入 | 结算月度商品销售 |
| 维护支出 | 扣除设施维护费用 |
| 董事会评估 | 评估目标完成度与满意度 |
| 青训刷新 | 刷新青年队可用球员 |
| 财务报告 | 生成月度财务报告 |

### 3.5 比赛日处理

```
processMatchDay()
  ├── 获取当日所有赛事
  ├── 对每场比赛执行 matchEngine.simulate()
  ├── 更新联赛积分榜
  ├── 更新球员统计（进球/助攻/评分）
  ├── 触发士气变动
  ├── 触发声望变动
  └── 生成比赛新闻
```

## 4. 球员系统

### 4.1 球员属性

球员拥有 **19项基础属性**，取值范围 1-20：

| 类别 | 属性 | 说明 |
|------|------|------|
| **身体** | speed | 速度 |
| | stamina | 耐力 |
| | strength | 力量 |
| | agility | 灵活性 |
| **技术** | passing | 传球 |
| | shooting | 射门 |
| | tackling | 抢断 |
| | dribbling | 盘带 |
| | defending | 防守 |
| **心理** | positioning | 位置感 |
| | vision | 视野 |
| | decisions | 决断 |
| | composure | 镇定 |
| | aggression | 侵略性 |
| | teamwork | 团队配合 |
| | leadership | 领导力 |
| **门将** | handling | 手控球 |
| | reflexes | 反应 |
| | aerial | 制空 |

### 4.2 附加属性

| 属性 | 范围 | 说明 |
|------|------|------|
| potential | 40-99 | 潜力值，决定成长上限 |
| ability | 20-99 | 当前能力值 |
| morale | 20-100 | 士气值 |
| fitness | 0-100 | 体能值 |
| age | 17-38 | 年龄 |

### 4.3 OVR(综合评分)计算

OVR 是球员在特定位置的综合实力评分，计算公式：

```
rawScore = baseScore * 0.40 + positionScore * 0.60

其中:
- baseScore = 所有19项属性的平均值
- positionScore = 该位置权重属性的加权平均值
```

**权重映射** (以 ST 前锋为例):

| 属性 | 权重 |
|------|------|
| shooting | 1.5 |
| speed | 1.3 |
| positioning | 1.2 |
| composure | 1.1 |
| strength | 1.0 |
| dribbling | 0.9 |

**分段线性映射** (rawScore → OVR):

| rawScore 区间 | 斜率 | 说明 |
|---------------|------|------|
| ≤ 13.0 | 5.0 | 低端球员，评分增长较快 |
| 13.0 - 15.5 | 6.5 | 中端球员，评分增长最快 |
| > 15.5 | 4.5 | 高端球员，评分增长放缓 |

**OVR 校准参照**:

| rawScore | OVR | 球员定位 |
|----------|-----|---------|
| 10.0 | 50 | 低级联赛球员 |
| 13.0 | 65 | 中游联赛球员 |
| 15.5 | 81 | 顶级联赛主力 |
| 18.0 | 92 | 世界级球星 |

### 4.4 球员身价计算

身价采用**指数增长模型**：

```
价值 = C × D^ovr × ageMult × potentialMult × reputationMult

其中:
- C = 4800 (基础常数)
- D = 1.15 (增长因子，每5点OVR身价翻倍)
```

**年龄修正系数 (ageMult)**:

| 年龄 | 系数 | 说明 |
|------|------|------|
| ≤ 22 | 1.25 | 年轻溢价 |
| 23-26 | 1.15 | 黄金年龄 |
| 27-29 | 1.00 | 当打之年 |
| 30-31 | 0.75 | 开始贬值 |
| 32-33 | 0.50 | 明显贬值 |
| ≥ 34 | 0.30 | 老将折扣 |

**潜力修正 (potentialMult)**: `1.0 + (potential - 70) * 0.005`

**声望修正 (reputationMult)**: `0.8 + reputation * 0.004` (reputation 范围 0-100)

**身价参照**:

| OVR | 基础身价 | 说明 |
|-----|---------|------|
| 60 | ~1.5M | 次级联赛球员 |
| 70 | ~6M | 中游主力 |
| 75 | ~12M | 强队轮换 |
| 80 | ~24M | 豪门主力 |
| 85 | ~49M | 顶级球星 |
| 90 | ~98M | 世界级巨星 |

### 4.5 球员特质

球员可获得**20种特质**，基于属性阈值自动解锁：

| 特质 | 触发条件 | 效果 |
|------|---------|------|
| Clinical (临床终结者) | shooting≥16, composure≥15 | 射门权重+35% |

| Leader (天生领袖) | leadership≥16, composure≥14 | 队友士气加成 |
| Speedster (闪电飞翼) | speed≥17, agility≥15 | 速度进攻权重提升 |
| Target Man (站桩中锋) | strength≥16, aerial≥15 | 制空对抗加成 |

## 5. 比赛引擎

### 5.1 模拟方式

采用**逐分钟模拟**，每场比赛模拟90分钟（含伤停补时）：

```
simulateMatch(homeTeam, awayTeam)
  ├── 计算双方战术强度
  ├── for minute = 1 to 90+补时 do
  │   ├── 计算本分钟进攻方
  │   ├── 判断是否产生射门机会
  │   ├── 判断射门是否进球
  │   ├── 选择进球球员
  │   ├── 选择助攻球员 (68%概率)
  │   ├── 判断黄/红牌
  │   ├── 判断受伤
  │   └── 更新球员体能
  └── 返回比赛结果
```

### 5.2 进球分配

进球数基于**泊松分布**，参数由双方战术对抗结果决定。

**射门球员选择权重**:

| 位置组 | 基础权重 |
|--------|---------|
| FWD (前锋) | 3.0 |
| CAM (前腰) | 2.0 |
| MID (中场) | 0.9 |
| DEF (后卫) | 0.2 |
| GK (门将) | 0.01 |

**特质加成**:
- Clinical/Poacher: 射门权重 +35%
- Playmaker: 助攻权重 +25%

### 5.3 助攻球员选择

- 68%概率产生助攻
- 权重属性: vision × 2.0 + passing × 1.5 + decisions × 1.0
- 排除进球球员本身

### 5.4 黄/红牌

基于 `aggression` 和 `decisions` 属性:
- 高侵略性 + 低决断 = 高犯规概率
- 累计2黄 = 红牌罚下

### 5.5 比赛伤病

- 比赛中有一定概率受伤
- 受伤概率受位置组和比赛风格影响
- 进攻型风格：前锋受伤风险更高
- 体能低时受伤概率上升

### 5.6 球员评分

每场比赛结束后为每位出场球员评分(1.0-10.0):
- 基础分 6.0-7.0
- 进球 +1.0-1.5
- 助攻 +0.5-0.8
- 关键传球 +0.3
- 被罚牌 -0.5/-1.0
- 失误 -0.5

## 6. 战术系统

### 6.1 阵型列表

游戏支持 **6大阵型**，每个阵型有 **3种变体**（共18种战术配置）:

| 阵型 | 变体A | 变体B | 变体C |
|------|-------|-------|-------|
| 4-4-2 | 经典 | 菱形中场 | 平行中场 |
| 4-3-3 | 进攻 | 防守 | 控制 |
| 3-5-2 | 进攻翼卫 | 防守 | 平衡 |
| 4-2-3-1 | 标准 | 宽幅 | 窄幅 |
| 5-3-2 | 防反 | 平衡 | 进攻 |
| 4-1-4-1 | 控制 | 防守 | 进攻 |

### 6.2 位置与角色

游戏定义 **13个位置**，每个位置有 **2-4种角色**:

| 位置 | 角色变体 |
|------|---------|
| GK (门将) | Goalkeeper, Sweeper Keeper |
| CB (中后卫) | Stopper, Cover, Ball-Playing |
| LB/RB (边后卫) | Defensive, Balanced, Wing-Back |
| CDM (后腰) | Anchor, Ball-Winner, Deep-Lying |
| CM (中场) | Box-to-Box, Deep-Lying, Mezzala |
| CAM (前腰) | Advanced, Shadow Striker, Trequartista |
| LM/RM (边前卫) | Wide, Inverted, Defensive |
| LW/RW (边锋) | Inside Forward, Traditional, Raumdeuter |
| ST (前锋) | Poacher, Target Man, False Nine, Complete |

### 6.3 比赛风格

| 风格 | 描述 | 效果 |
|------|------|------|
| Balanced | 平衡攻守 | 基准值 |
| Attacking | 进攻优先 | 进攻+, 防守- |
| Defensive | 防守优先 | 防守+, 进攻- |
| Possession | 控球为主 | 控球率+, 反击- |
| Counter | 防守反击 | 反击+, 控球- |
| HighPress | 高位逼抢 | 前场压迫+, 体能消耗+ |

### 6.4 战术对抗

战术之间存在**克制关系**:
- High Press 克制 Possession（高压打乱节奏）
- Counter 克制 Attacking（防反杀对手进攻）
- Possession 克制 Counter（控球压制反击）
- Balanced 无明显克制/被克制

## 7. 转会系统

### 7.1 转会窗口

| 窗口 | 时间 | 说明 |
|------|------|------|
| 夏季窗口 | 6月-8月 | 主要转会期 |
| 冬季窗口 | 1月 | 补充转会期 |
| 窗口外 | 其他时间 | 仅可签自由球员 |

### 7.2 转会流程

```
发起报价
  ├── 对方俱乐部评估
  │   ├── 接受 → 与球员谈判
  │   │   ├── 球员接受 → 转会完成
  │   │   └── 球员拒绝 → 转会失败 (7天冷却)
  │   └── 拒绝 → 可提高报价 (7天冷却)
  └── 超出窗口 → 转会失败
```

### 7.3 报价评估因素

| 因素 | 影响 |
|------|------|
| 出价/身价比 | 越高越易接受 |
| 球员重要性 | 核心球员溢价更高 |
| 合同剩余时长 | 合同短→要价低 |
| 球员意愿 | 球员想离开→降低要求 |
| 俱乐部关系 | 友好关系→更易谈判 |

### 7.4 其他转会方式

| 方式 | 说明 |
|------|------|
| 自由转会 | 合同到期球员，任何时间可签 |
| 租借 | 临时借用球员 (6-12个月) |
| 预签约 | 合同最后6个月可提前签约 |
| 球员交换 | 球员+现金交换 |

### 7.5 AI转会行为

- AI每周处理转会逻辑
- 转会窗口期间，周四额外处理一次
- AI会根据球队需要、预算、球员年龄等因素决策
- 5%的日概率触发转会传闻新闻

### 8.2 门票系统

**票价策略**:

| 策略 | 票价倍率 | 上座率修正 |
|------|---------|-----------|
| 低价 (low) | 0.70× | +12% |
| 标准 (standard) | 1.00× | 0% |
| 高价 (high) | 1.40× | -8% |
| 溢价 (premium) | 1.80× | -18% |

**上座率影响因素**:
- 球队成绩（联赛排名）
- 球队声望
- 对手实力
- 票价策略
- 天气/赛事重要性

**智能上座率算法**:
- 基于球队实力对比动态调整
- 德比战/强强对话自动提升上座率
- 连续输球降低上座率

## 10. 士气系统

### 10.1 士气等级

| 等级 | 士气范围 | 描述 | 表现修正 |
|------|---------|------|---------|
| 5 | ≥ 80 | 斗志昂扬 | +5% |
| 4 | 60-79 | 状态良好 | 0% |
| 3 | 40-59 | 情绪一般 | -2% |
| 2 | 20-39 | 心生不满 | -5% |
| 1 | < 20 | 极度低落 | -10% |

### 10.2 影响因素

**正面影响** (提升士气):
- 赢球（尤其大胜）
- 进球/助攻/高评分
- 正面新闻
- 团队谈话效果好
- 获得新合同
- 球队连胜

**负面影响** (降低士气):
- 输球（尤其惨败）
- 连续不上场
- 被挂牌出售
- 合同谈判破裂
- 队内核心离开
- 球队连败

### 10.3 士气核心 (Morale Core)

每个球员有深层士气属性:

| 属性 | 说明 |
|------|------|
| manager_trust | 对主教练的信任度 |
| unresolved_issue | 未解决的不满 |
| talk_fatigue | 谈话疲劳（防止频繁谈话） |

### 10.4 团队谈话

主教练可以发起团队谈话来影响士气:
- 每次谈话有冷却时间
- 效果取决于球员性格和与教练的关系
- 过于频繁的谈话效果递减

## 11. 董事会系统

### 11.1 声望分档与目标

董事会根据俱乐部声望设定赛季目标:

| 声望分档 | 声望范围 | 典型目标 |
|----------|---------|---------|
| 精英 (elite) | ≥ 80 | 夺冠 / 联赛前2 |
| 强队 (strong) | 65-79 | 联赛前4 / 欧冠资格 |
| 中游 (mid) | 45-64 | 联赛前半区 |
| 弱队 (weak) | 25-44 | 保级 / 下半区 |
| 最弱 (lowest) | < 25 | 避免垫底 |

### 11.2 满意度评估

每月1号进行董事会满意度评估:
- 比较当前联赛排名与目标排名
- 高于预期 → 满意度上升
- 低于预期 → 满意度下降
- 大幅低于预期 → 发出警告

### 11.3 解雇机制

| 阶段 | 条件 | 后果 |
|------|------|------|
| 警告 | 满意度低于阈值 | 收到董事会警告邮件 |
| 最后通牒 | 连续低满意度 | 限期改善 |
| 解雇 | 未能改善 | 游戏结束 (可选重新就业) |

### 11.4 董事会请求

玩家可以向董事会提出请求:
- 申请注资（每赛季2次限额）
- 申请扩建球场
- 申请升级设施

请求成功率受满意度影响。
