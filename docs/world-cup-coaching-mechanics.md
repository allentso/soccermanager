## 1. 机制概述

世界杯执教是游戏中的**双重身份系统**——玩家在管理俱乐部的同时，可以在世界杯年兼任一支国家队的主教练，带队征战世界杯。

**核心设计理念**：
- 每 4 年一次的稀有事件，增加长期游玩的目标感
- 声望驱动的邀请机制，奖励长期经营
- 不干扰俱乐部赛季的主线进程（世界杯安排在赛季初的 6-7 月）
- 国家队阵容基于游戏内真实球员数据，而非独立数据库

**身份切换**：
```
gameState.currentRole = "club"          -- 俱乐部模式（默认）
gameState.currentRole = "national_team" -- 国家队模式（世界杯期间）
```

## 2. 触发条件与时间线

### 2.1 世界杯年判定

```lua
-- 首届世界杯: 2026（美加墨）
-- 周期: 每4年
FIRST_WORLD_CUP = 2026
CYCLE = 4

-- 判定公式
isWorldCupYear = (year - 2026) % 4 == 0
-- 世界杯年份: 2026, 2030, 2034, 2038, ...
```

### 2.2 赛季日历调整

世界杯年的赛季日历会自动调整：

| 类型 | 赛季开始日期 | 说明 |
|------|------------|------|
| 普通赛季 | 8月10日 | 标准联赛开始 |
| 世界杯赛季 | 6月1日 | 提前开始，先踢世界杯 |

### 2.3 完整时间线

```
赛季末 → SeasonManager.endSeason()
  ↓
新赛季初始化 → _startNewSeason()
  ↓
WorldCup.initialize(gameState)
  ↓
分组抽签 + 发送国家队邀请
  ↓
玩家通过收件箱接受/拒绝邀请
  ↓
（接受后）进入大名单选拔页面
  ↓
6月12日-28日：小组赛
  ↓
6月30日-7月4日：32强赛
  ↓
7月5日-8日：16强赛
  ↓
7月10日-12日：1/4决赛
  ↓
7月15日-16日：半决赛
  ↓
7月19日：季军赛 / 7月20日：决赛
  ↓
世界杯结束 → 自动解除国家队任期 → 回归俱乐部
```

## 3. 国家队邀请系统

### 3.1 声望门槛

玩家经理声望（1-99 量表）必须达到 **40** 才有资格收到邀请。

```lua
NT_REP_THRESHOLD = 40
```

**声望等级参考**：
| 声望范围 | 称号 | 能否收到邀请 |
|---------|------|------------|
| 1-24 | 无名之辈 | 不能 |
| 25-39 | 新人教练 | 不能 |
| 40-54 | 普通教练 | 可以（C档国家） |
| 55-69 | 知名教练 | 可以（B/C档） |
| 70-84 | 名帅 | 可以（A/B/C档） |
| 85-99 | 传奇教头 | 可以（全部档位） |

### 3.2 国家队分档

国家队按实力分为 4 个档位：

| 档位 | 国家 | 说明 |
|------|------|------|
| **S档** | 巴西、法国、阿根廷、英格兰、西班牙、德国 | 顶级强队 |
| **A档** | 葡萄牙、荷兰、比利时、乌拉圭、克罗地亚、哥伦比亚 | 传统强队 |
| **B档** | 墨西哥、美国、日本、韩国、瑞士、塞内加尔、摩洛哥等13支 | 中上球队 |
| **C档** | 捷克、挪威、厄瓜多尔、伊朗、突尼斯、加纳等23支 | 中等球队 |

### 3.3 声望 → 可邀约档位映射

| 经理声望 | 可收到邀约的档位 |
|---------|----------------|
| ≥ 70 | S + A + B + C（全部） |
| ≥ 55 | A + B + C |
| ≥ 40 | B + C |
| ≥ 25 | C |

### 3.4 邀约概率计算

每个候选国家的邀约概率：

```lua
-- 基础概率（按档位）
TIER_PROBABILITY = { S = 0.50, A = 0.40, B = 0.30, C = 0.25 }

-- 声望加成
repBonus = min(0.30, (rep - 40) / 500)

-- 最终概率
finalProb = min(0.85, baseProb + repBonus)
```

### 3.5 邀约规则

1. **保底邀约**：玩家所在联赛对应的国家队（如在英超执教→英格兰）必定发出邀请（前提是该国入围世界杯）
2. **最大邀约数**：同时最多收到 **4** 份邀请
3. **2030+玩家国家保护**：如果玩家已接受国家队，该国必定入围世界杯
4. **只能选择1支**：玩家接受一支后，其余自动失效

### 3.6 联赛 → 国家映射

| 联赛 | 对应国家（保底邀约） |
|------|-------------------|
| 英超 (premier_league) | 英格兰 (ENG) |
| 西甲 (la_liga) | 西班牙 (ESP) |
| 德甲 (bundesliga) | 德国 (GER) |
| 意甲 (serie_a) | 意大利 (ITA)* |
| 法甲 (ligue_1) | 法国 (FRA) |

> *注：意大利未在2026世界杯参赛国列表中，不会作为保底邀约生效。

## 4. 大名单选拔

### 4.1 球员来源

国家队阵容**不是独立数据库**，而是从游戏中所有俱乐部球员中筛选：

```lua
-- 筛选条件
player.nationality == toPlayerNat(nationFifaCode)  -- 国籍匹配
player.retired == false                             -- 未退役
player.injured == false                             -- 未受伤
```

### 4.2 虚拟球员补位

当某国籍球员不足 23 人时，系统会自动生成虚拟球员补位：

- 能力值范围按国家档位决定：
  - S档：68-82 OVR
  - A档：62-76 OVR
  - B档：55-70 OVR
  - C档：48-64 OVR
- 位置分配根据现有球员的缺口自动分配
- 生成带国籍风格的随机姓名

### 4.3 大名单规则

| 规则 | 数值 |
|------|------|
| 大名单人数 | 23人 |
| 最少门将数 | 3人 |
| 最少确认人数 | 11人（才可确认） |

### 4.4 选拔流程

1. 玩家接受邀请后，系统自动生成**默认23人名单**（按 OVR 排序，确保3名门将）
2. 玩家进入 `national_squad_select` 页面，可按位置筛选调整
3. 点击球员行切换选中/取消状态
4. 确认后锁定名单（`squadConfirmed = true`），不可再修改
5. 未确认的情况下，系统仍按当前选择出战

### 4.5 阵型与风格

```lua
-- 默认阵型: 4-3-3
-- 默认风格: Balanced
-- 可通过 _nationalTeamSettings[nationCode] 自定义
```

## 5. 比赛模拟

### 5.1 国家队强度计算

国家队实力**动态计算**，基于当前在役球员的 overall 属性平均值：

```lua
-- TurnProcessor._simulateWCMatch() 中的逻辑
-- 扫描 gameState.players 中所有匹配国籍的球员
-- 取前 N 个球员的 overall 平均值作为国家队强度
```

### 5.2 比赛引擎

世界杯比赛使用**自定义 Poisson 分布引擎**（`TurnProcessor._simulateWCMatch()`），区别于联赛使用的标准 `MatchEngine`：

- 基于国家队强度差异计算期望进球数
- 淘汰赛平局自动进入加时赛 + 点球大战
- 点球大战结果记录在 `fixture._penaltyWinner`

### 5.3 玩家国家队比赛

当比赛涉及玩家执教的国家队时：
- `WorldCup.isPlayerNationMatch()` 返回 `true`
- 比赛标记为 `_pendingPlayerMatch = true`
- 进入玩家可操控的 `pre_match` → `match_live` 流程
- 玩家可选择首发阵容（从已选大名单中挑选11人）

### 5.4 首发阵容自动选择

AI 国家队和默认首发使用位置优先的贪心算法：

```
目标: 1 GK + 4 DEF + 3 MID + 3 FWD = 11人
算法: 每个位置组按 overall 降序取满所需人数
不足: 从剩余球员中按 overall 补充
```

## 6. 赛程与日历集成

### 6.1 2026 真实分组（首届固定）

48支球队分为12组（A-L），每组4队。首届使用真实的2026美加墨世界杯分组。

### 6.2 2030+动态抽签

从第二届开始，参赛国家通过以下方式产生：
1. 16支种子队必定入围
2. 玩家执教的国家（如有）必定入围
3. 从33支候补队中随机抽取填满48支
4. 48支队伍随机分入12组

### 6.3 赛程日期

| 阶段 | 日期范围 | 场次 |
|------|---------|------|
| 小组赛 | 6月12日 - 6月28日 | 72场（每组6场） |
| 32强赛 | 6月30日 - 7月4日 | 16场 |
| 16强赛 | 7月5日 - 7月8日 | 8场 |
| 1/4决赛 | 7月10日 - 7月12日 | 4场 |
| 半决赛 | 7月15日 - 7月16日 | 2场 |
| 季军赛 | 7月19日 | 1场 |
| 决赛 | 7月20日 | 1场 |

### 6.4 出线规则

```
小组赛（每组4队，共12组）
  ↓
每组前2名（24队）+ 8个最佳第三名 = 32队
  ↓
32强赛（16场，单场淘汰）
  ↓
16强赛（8场，单场淘汰）
  ↓
1/4决赛 → 半决赛 → 决赛
```

### 6.5 最佳第三名排序规则

```
积分 > 净胜球 > 进球数
```

### 6.6 淘汰赛平局处理

淘汰赛阶段平局 → 点球大战：
- AI对AI：随机决定胜者
- 涉及玩家国家队：`_penaltyWinner` 由比赛引擎判定

## 7. 奖励与影响

### 7.1 直接奖励

| 成就 | 奖励 |
|------|------|
| 带队夺冠 | 收件箱高优先级消息 + 历史记录 |
| 世界杯冠军记录 | `RecordsManager.onWorldCupChampionship()` |
| 冠军历史 | `HistoryManager.recordWorldCupChampion()` |

### 7.2 声望影响

世界杯期间的比赛结果会通过 `ReputationManager` 影响：
- 球队声望：胜 +2.5，负 -1.0，大胜(3+) +1.5 额外
- 经理声望：球队变化 / 5

### 7.3 对俱乐部的影响

- 世界杯期间俱乐部日常事务**正常运转**（训练、伤病恢复、合同等）
- 参加世界杯的俱乐部球员不会受到额外疲劳惩罚（当前版本简化处理）
- 世界杯不影响俱乐部的联赛赛程（联赛在世界杯结束后才开始）

## 8. 任期结束机制

### 8.1 自动解除

世界杯结束时（`_completeTournament`），国家队教练身份**自动解除**：

```lua
-- 世界杯完赛时自动执行
gameState.nationalTeamCoach = nil
gameState.currentRole = "club"
```

### 8.2 解除消息

根据最终成绩发送不同消息：
- **夺冠**："恭喜！你带领X国夺得世界杯冠军！国家队任期已圆满结束。"
- **未夺冠**："世界杯已结束，你的X国主教练任期已到期。感谢你的付出。"

### 8.3 不可主动辞职

当前版本不支持世界杯期间主动辞去国家队教练职务——一旦接受，必须完成整届赛事。

## 9. 数据结构

### 9.1 GameState 相关字段

```lua
gameState.worldCup = {
    -- Tournament 对象
    name = "世界杯",
    type = "world_cup",
    season = 2026,
    phase = "group" | "r32" | "r16" | "qf" | "sf" | "final" | "completed",
    qualifiedTeams = {"FRA", "BRA", ...},  -- 48个FIFA代码
    groups = {
        A = { teamIds = {...}, standings = {...}, fixtures = {...} },
        ...
    },
    knockout = {
        r32 = { fixtures... },
        r16 = { fixtures... },
        qf  = { fixtures... },
        sf  = { fixtures... },
        final = { fixtures... },  -- 包含季军赛和决赛
    },
    champion = "FRA",  -- 冠军国家代码（赛事结束后设置）
}

gameState.nationalTeamCoach = {
    nation = "ENG",           -- 执教的国家FIFA代码
    squad = {101, 203, ...},  -- 大名单球员ID列表
    squadConfirmed = true,    -- 是否已确认锁定
}

gameState.currentRole = "club" | "national_team"

gameState._nationalTeamSettings = {
    ENG = { formation = "4-3-3", playStyle = "Balanced" },
    ...
}
```

### 9.2 比赛 Fixture 结构（世界杯）

```lua
{
    id = "r16_3",
    groupName = "A",           -- 仅小组赛有
    round = 1,                 -- 小组赛轮次
    leg = 1,                   -- 固定为1（单场制）
    matchIndex = 3,
    homeTeamId = "FRA",        -- FIFA国家代码
    awayTeamId = "BRA",
    date = {year = 2026, month = 7, day = 6},
    status = "scheduled" | "finished",
    homeGoals = 2,
    awayGoals = 1,
    isKnockout = true,         -- 淘汰赛标志
    _isThirdPlace = false,     -- 季军赛标志
    _penaltyWinner = "FRA",    -- 点球大战胜者（平局时）
}
```

### 9.3 FIFA代码 → 球员国籍映射

世界杯使用 FIFA 代码（如 `CRO`），球员数据使用 ISO 或自定义代码（如 `HR`）。转换通过 `FIFA_TO_PLAYER_NAT` 表完成：

```lua
FIFA_TO_PLAYER_NAT = {
    CRO = "HR",    -- 克罗地亚
    GER = "DE",    -- 德国
    ENG = "ENG",   -- 英格兰（特殊：保持不变）
    FRA = "FR",    -- 法国
    BRA = "BR",    -- 巴西
    ARG = "AR",    -- 阿根廷
    -- ... 等
}
```

## 10. 已知限制与未来扩展方向

### 10.1 当前限制

| 限制 | 说明 |
|------|------|
| 无预选赛 | 参赛队直接由种子+随机决定，无真实预选赛模拟 |
| 无球员疲劳联动 | 世界杯参赛球员回到俱乐部后无疲劳/受伤风险加成 |
| 无独立国家队阵容 | 依赖俱乐部球员数据，无法召唤"退役后复出"球员 |
| 不可中途辞职 | 接受后必须完成整届 |
| 无经济收入 | 国家队执教无额外薪资或奖金 |
| 无战术深度 | 国家队比赛使用简化的 Poisson 引擎 |
| 无续任机制 | 每届世界杯都需要重新接受邀请 |
| 无友谊赛/热身赛 | 世界杯前无热身赛集训期 |

### 10.2 可能的扩展方向

1. **预选赛系统**：在世界杯前1-2年加入洲际预选赛，玩家需带队出线
2. **球员征召冲突**：世界杯期间俱乐部无法使用被征召球员
3. **国家队声望独立系统**：累积国家队执教声望，影响下届邀约
4. **经济奖励**：世界杯奖金分成，可转入俱乐部或个人
5. **多届续任**：连续表现好可自动续任，无需等待邀请
6. **热身赛/友谊赛**：世界杯前的集训和热身赛
7. **球员状态联动**：世界杯参赛球员赛后疲劳/受伤概率增加
8. **洲际杯赛**：欧洲杯、美洲杯等区域性赛事
9. **U23国家队/奥运会**：青年国家队执教选项
10. **助教系统**：可指定助教自动管理国家队日常训练
