## 1. 游戏概述

### 1.1 高层概念

OpenFoot Manager 是一款足球经理模拟游戏。玩家扮演一支职业足球俱乐部的主教练，负责球队管理的方方面面：战术制定、阵容选择、球员转会、青训培养、财务管理等。游戏以回合制（逐日推进）方式进行，玩家通过决策引领球队走向荣耀。

### 1.2 核心特点

- **真实数据**: 基于 FM2024 数据，涵盖欧洲五大联赛 98 支球队、2000+ 名真实球员
- **深度模拟**: 19 项球员属性、战术系统、比赛引擎、转会市场、财务体系
- **多赛事体系**: 国内联赛 + 欧冠联赛 + 世界杯（2026新赛制）
- **长期经营**: 球员成长/衰退、青训培养、潜力系统、职业生涯记录
- **求职系统**: 被解雇后可申请新职位或接受邀约，体验完整教练生涯

### 1.3 游戏起始

- **赛季起始日期**: 2025年8月10日
- **初始流程**: 主菜单 → 创建经理（填写姓名、国籍、出生年份）→ 选择执教球队 → 进入游戏主界面

## 2. 核心玩法循环

### 2.1 回合制时间推进

游戏以"天"为最小时间单位逐步推进。玩家在完成当日决策后点击"下一天"按钮推进时间。

```
┌─────────────────────────────────────────────────┐
│                  每日循环                         │
├─────────────────────────────────────────────────┤
│  比赛日:                                         │
│    → 模拟所有联赛/杯赛比赛                        │
│    → 更新积分/排名/统计                           │
│    → 更新士气/声望                               │
│    → 比赛收入结算                                │
│                                                 │
│  非比赛日:                                       │
│    → 训练效果结算                                │
│    → 伤病恢复                                    │
│    → 体能恢复                                    │
│    → 合同事务处理                                │
│    → 转会市场处理                                │
│    → 球探报告生成                                │
│    → 青训更新                                    │
│    → 随机事件触发                                │
├─────────────────────────────────────────────────┤
│  每周（周一）:                                    │
│    → 工资发放                                    │
│    → 伤病判定                                    │
│    → 士气周更新                                  │
│    → AI管理决策                                  │
│    → 新闻生成                                    │
├─────────────────────────────────────────────────┤
│  每月（1号）:                                     │
│    → 赞助商/转播/商品收入                         │
│    → 维护费用支出                                │
│    → 董事会评估                                  │
│    → 青训池刷新                                  │
│    → 声望衰减/回归                               │
└─────────────────────────────────────────────────┘
```

### 2.2 玩家核心决策

| 决策类别 | 内容 |
|---------|------|
| **阵容管理** | 选择首发11人、替补、设定阵型 |
| **战术制定** | 选择踢球风格、进攻模式、定位球主罚人 |
| **转会运营** | 买入/卖出球员、处理报价、球探调配 |
| **训练安排** | 团队训练重点、个人训练、训练强度 |
| **青训培养** | 招募青年球员、提拔一线队 |
| **合同谈判** | 续约球员、终止合同 |
| **财务管理** | 设定转会预算/工资预算 |
| **赛前动员** | 赛前鼓舞士气 (Team Talk) |

## 3. 游戏世界

### 3.1 联赛体系

游戏包含欧洲五大联赛：

| 联赛 | 球队数 | 国家 | 缩写 |
|------|-------|------|------|
| 英格兰超级联赛 | 20 | ENG | EPL |
| 西班牙甲级联赛 | 20 | ESP | LaLiga |
| 意大利甲级联赛 | 20 | ITA | SerieA |
| 德国甲级联赛 | 18 | GER | Bundesliga |
| 法国甲级联赛 | 18 | FRA | Ligue1 |

**总计**: 96 支球队，2000+ 球员

### 3.2 联赛赛制

- **双循环**: 每支球队与其他所有球队各交手两次（主场 + 客场）
- **积分规则**: 胜3分、平1分、负0分
- **排名规则**: 积分 > 净胜球 > 进球数 > 对赛成绩
- **比赛日**: 对齐到周末（周六/周日）

### 3.3 日期系统

- 赛季从8月10日开始
- 冬季转会窗：1月
- 夏季转会窗：6-8月
- 赛季约在次年5月结束

| **技术** | 传球 (passing) | 短传长传精度 |
| | 射门 (shooting) | 射门力度和准度 |
| | 抢断 (tackling) | 断球能力 |
| | 盘带 (dribbling) | 控球过人 |
| **防守** | 防守 (defending) | 防守意识 |
| | 位置感 (positioning) | 站位跑位 |
| **心理** | 视野 (vision) | 传球视野 |

| | 制空 (aerial) | 高空球处理 |

### 4.2 综合能力 (OVR) 计算

```
OVR = 40% × 基础分 + 60% × 位置加权分

基础分 = 所有属性平均值
位置加权分 = 根据球员位置的关键属性加权

最终映射（分段线性）:
- 属性均值 10 → OVR 50
- 属性均值 14 → OVR 70  
- 属性均值 17 → OVR 85
- 属性均值 20 → OVR 99
```

### 4.5 潜力系统 (PA Rating)

每个球员有潜力评级 (PA Rating)，范围 1.0-10.0（步进0.5）：

| PA Rating | 描述 | 说明 |
|-----------|------|------|
| 10.0 | 传奇巨星 | 足坛历史级别 |
| 9.0-9.5 | 世界巨星 | 金球奖级别 |
| 8.0-8.5 | 洲际球星 | 顶级联赛核心 |
| 7.0-7.5 | 国家队主力 | 国脚级别 |
| 6.0-6.5 | 联赛球星 | 联赛最佳阵容 |
| 5.0-5.5 | 优秀球员 | 稳定首发 |
| 4.0-4.5 | 良好球员 | 轮换/替补 |
| 3.0-3.5 | 普通球员 | 板凳深度 |
| 2.0-2.5 | 一般球员 | 低级别联赛 |
| 1.0-1.5 | 无潜力 | 业余水平 |

**潜力实现机制**:
- 每次游戏开始时，基于 PA Rating 生成"实际潜力"
- 高评级球员方差更小（确定性高），低评级球员方差更大
- 实际潜力 = 中心值 ± 高斯随机偏移

### 4.6 球员成长与衰退

- **成长期**: < 24岁，属性有机会通过训练增长
- **巅峰期**: 24-29岁，属性稳定
- **衰退期**: > 30岁，身体属性逐渐下降
- **退役**: 赛季结束时，36岁以上球员有概率退役

### 4.7 特质系统

球员拥有自动计算的特质标签（基于属性阈值）：

| 特质 | 条件 | 比赛效果 |
|------|------|---------|
| 飞毛腿 | 速度≥17, 敏捷≥14 | - |
| 铁人 | 耐力≥17, 力量≥14 | - |
| 传球大师 | 传球≥17, 视野≥15 | - |
| 射术精湛 | 射门≥17, 镇定≥14 | +35%射门权重 |
| 防守铁闸 | 抢断≥17, 防守≥16 | - |
| 盘球艺术家 | 盘带≥17, 敏捷≥15 | - |
| 领袖气质 | 领导力≥16, 镇定≥14 | - |
| 空霸 | 力量≥16, 制空≥16 | - |
| ... | ... | ... |

共 20 种自动特质。

### 4.8 球员身价计算

```
基础价值 = D^(OVR-50) × 基准价
  D = 1.15（每1点OVR差距 = 15%价值差）

乘数修正:
  × 年龄乘数（年轻球员溢价，30+岁折扣）
  × 潜力溢价（高PA +20%~40%）
  × 声望修正（知名度加成）
```

| rotation | 轮换球员 |
| squad | 替补阵容 |
| youth | 青年球员 |
| loaned | 外租球员 |

## 6. 比赛模拟

### 6.1 模拟引擎概述

比赛引擎以**分钟**为单位模拟 90 分钟（含补时）的比赛。基于球队整体实力、战术设定、主场优势等因素计算比赛结果。

### 6.2 进球模型

使用**泊松分布**（Poisson Distribution）计算期望进球数：

```
期望进球 = f(球队攻击力, 对手防守力, 战术修正, 主场优势)

实际进球分布: P(k) = (λ^k × e^(-λ)) / k!
  其中 λ = 期望进球数
```

### 6.3 事件权重

**进球球员选择** (加权随机):
- 前锋 (FWD): 权重 ×3.0
- 前腰 (CAM): 权重 ×2.0
- 中场 (MID): 权重 ×1.0
- 后卫 (DEF): 权重 ×0.3
- 门将 (GK): 权重 ×0.01

**助攻球员选择**:
- 前腰 (CAM): 权重 ×1.8
- 中场 (MID): 权重 ×1.35
- 边锋 (LW/RW): 权重 ×1.5
- 后卫 (DEF): 权重 ×0.5

**黄/红牌选择**:
- 防守型球员 + 高侵略性属性: 更高概率

### 6.4 伤病事件

- 比赛中有概率发生伤病
- 高逼抢/高强度风格增加受伤概率
- 伤病按位置组有不同风险分布
- 伤病严重程度: 1-90天不等

### 6.5 特质加成

特定球员特质在比赛中提供额外效果:
- "射术精湛" → 进球权重 +35%
- "传球大师" → 助攻权重 +25%
- "飞毛腿" → 反击模式额外进攻加成

### 6.6 主场优势

主队获得约 +10% 的进攻/士气加成。

### 6.7 比赛结果影响

每场比赛后:
- 更新联赛积分榜
- 更新球员数据统计（进球、助攻、黄牌等）
- 更新球队/经理声望
- 影响球员士气
- 影响球员体能/伤病

## 7. 转会系统

### 7.1 转会窗口

| 窗口 | 时间 | 说明 |
|------|------|------|
| 夏季转会窗 | 6月-8月 | 主要转会期 |
| 冬季转会窗 | 1月 | 补强窗口 |

### 7.2 转会流程

```
发现球员（球探报告/直接查询）
  ↓
提交报价（设定金额）
  ↓
对方俱乐部响应（接受/拒绝/还价）
  ↓
与球员谈判薪资合同
  ↓
完成转会 / 交易失败
```

### 7.3 转会规则

- **拒绝冷却期**: 报价被拒后需等待 7 天才能再次报价
- **预合同锁定**: 避免重复对同一球员操作
- **分期付款**: 大额转会支持分期付款
- **租借**: 支持球员外租（赛季结束自动归还）
- **自由球员**: 合同到期球员可免费签约

### 7.4 AI 转会行为

AI 球队也会:
- 主动买入/卖出球员
- 处理租借到期
- 进行合理的市场操作
- 基于球队需求和预算做决策

### 7.5 球探系统

- 可派遣球探关注特定球员
- 球探报告需要时间生成
- 球探评估球员能力/潜力
- 球探设施等级影响报告质量和速度

## 14. 合同系统

- **周薪**: 球员的每周工资
- **合同年限**: 1-5年
- **到期日**: year + month

### 14.2 合同到期提醒

| 剩余月数 | 提醒级别 |
|---------|---------|
| 6个月 | 普通提醒 |
| 3个月 | 紧急警告 |
| 0 | 球员自动离队 |

### 14.3 续约机制

续约成功率取决于:
- 薪资涨幅（涨薪50%+ → +30%成功率）
- 球队声望（80+ → +15%）
- 球员年龄（32+ → +15%，年轻人更挑剔）
- 合同年限合理性

### 14.4 终止合同

- 支付赔偿金 = 剩余合同薪资 × 50%
- 球员立即成为自由球员

## 17. 赛季流程

### 17.1 赛季结束处理顺序

```
1. 目标评估 → 判断赛季目标是否完成
2. 董事会评估 → 满意度变动
3. 奖金发放 → 根据排名发放赛季奖金
4. 升降级 → 联赛排名调整（预留功能）
5. 赛季最佳 → 选出联赛最佳球员/射手/助攻/门将/青年
6. 球员发展 → 年轻球员成长、老球员衰退
7. 特质重算 → 根据新属性重新计算特质
8. 合同到期 → 释放到期球员
9. 退役判定 → 36+岁球员概率退役
10. 职业历史 → 记录赛季表现到球员/经理履历
11. 记录更新 → 更新队史射手榜等
12. 统计重置 → 清空赛季个人数据
13. 历史归档 → 写入世界历史记录
14. 新赛季初始化 → 生成新赛季联赛赛程、重置积分榜
```

### 17.2 赛季最佳奖项

| 奖项 | 评选标准 |
|------|---------|
| 联赛最佳球员 | 综合 OVR + 赛季表现 |
| 最佳射手 | 赛季进球数 |
| 最佳助攻 | 赛季助攻数 |
| 最佳门将 | 扑救 + 零封场次 |
| 最佳新秀 | 23岁以下最佳表现 |

## 20. 求职系统

### 20.1 失业状态

当玩家被解雇后:
- 进入失业状态
- 可浏览空缺职位
- 可主动申请职位
- 会收到球队主动邀约

### 20.2 主动申请

- **冷却期**: 申请被拒后等待 7 天
- **成功率**: 基于经理声望 vs 球队声望差距

| 声望差 (球队-经理) | 成功率 |
|-------------------|--------|
| >30 | 10% |
| 20-30 | 25% |
| 10-20 | 40% |
| 0-10 | 55% |
| <0 (经理更高) | 75% |

### 20.3 主动邀约

- **生成概率**: 15%基础 + 每失业1天增加2%（上限60%）
- **邀约冷却**: 每次邀约间隔 5 天
- **过期时间**: 7 天未回复自动过期
- **最多持有**: 3 个未处理邀约
- **匹配逻辑**: 球队声望不超过经理声望+40

### 20.4 AI填补空缺

- 球队空缺 14 天后自动由AI填补
- AI优先雇佣失业的AI经理（声望匹配）
- 无合适人选时生成新经理

### 20.5 随机新空缺

- 每天 0.3% 概率产生新的教练空缺
- 模拟AI教练辞职/合同到期
- 玩家失业时收到新空缺通知

### 25.2 设计模式

| 模式 | 应用 |
|------|------|
| **事件总线** (EventBus) | 模块间解耦通信 |
| **路由器** (Router) | UI页面导航管理 |
| **数据模型** (Domain Models) | 实体的面向对象封装 |
| **管理器** (Manager Pattern) | 各系统独立管理器，单一职责 |
| **序列化** (Serialize/Deserialize) | 游戏状态的保存/加载 |
| **回合处理** (Turn Processor) | 统一的每日/周/月处理流程 |

### 25.3 关键技术特点

- **事件驱动**: 通过 EventBus 发送消息，降低模块耦合
- **纯数据模拟**: 无3D渲染，全部基于数据计算
- **确定性随机**: 使用引擎 Random/RandomInt，可通过种子复现
- **增量更新**: 每日处理只计算当天变化，不全量重算
- **延迟加载**: FM数据在新游戏创建时一次性加载
